# Admin Preset 与 Basic RBAC

本文档定义当前模板的两种初始化形态，避免把后台项目的通用能力强行变成所有派生项目的默认负担。

## 初始化形态

| 形态 | 适用项目 | 默认保留 | 初始化建议 |
| --- | --- | --- | --- |
| base | 官网、用户端、小工具、API 服务 | 用户登录 / 注册、通用鉴权、错误码、健康检查、可观测性、质量门禁 | 按需裁掉管理员后台页面和 admin preset 文案 |
| admin preset | 管理台、运营后台、内部系统 | antd 后台 Layout、管理员登录、账号目录、角色权限概览、basic RBAC 服务端校验 | 替换菜单、项目名、默认管理员和业务模块 |

当前仓库主干保留 admin preset 的最小可用骨架，但业务字段、行业页面、复杂权限策略仍由派生项目按真实需求扩展。

## Basic RBAC 边界

服务端权限是真源，前端权限只负责菜单、路由和按钮显示。

当前内置权限码：

| 权限码 | 含义 |
| --- | --- |
| `admin.access` | 允许进入后台基础入口 |
| `admin.user.read` | 允许查看账号目录 |
| `admin.user.write` | 允许启用或禁用普通用户账号 |
| `admin.rbac.read` | 允许查看角色权限基线 |

默认内置角色：

| 角色 | 含义 |
| --- | --- |
| `super_admin` | 模板内置最高权限角色，初始化管理员默认绑定 |

启动初始化会幂等写入默认角色、权限码、角色权限绑定，并给配置中的默认管理员绑定 `super_admin`。已有管理员也会补齐该绑定，避免迁移后旧账号失权。

## 明确不内置的能力

以下能力不属于模板通用基线：

- 多租户
- 部门树 / 组织架构
- 数据权限
- 字段级权限
- 审批流权限
- 行业业务角色
- 动态权限表达式

派生项目确实需要时，应在自己的仓库新增 schema、接口、前端页面、测试和文档，不要把项目特例反写回模板通用层。

## 前端边界

- antd 只用于后台 preset 页面。
- 用户端、首页、登录注册页不因后台 preset 切换设计体系。
- 后台菜单与按钮可以根据 `auth.admin_login` / `auth.me` 返回的 `permissions` 做显示控制。
- 前端隐藏不等于授权，所有关键操作必须继续由服务端校验权限码。
